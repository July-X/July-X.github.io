## 行走摆动：一个 sin 函数搞定

玩家飞船的"行走"动画是 `feedback_controller.gd` 里的一个 `_walk_cycle` 计数器：

```python
var _walk_cycle: float = 0.0

func _update_walk_animation(delta: float) -> void:
    if _player.velocity.length() > 10.0:
        _walk_cycle += delta * 8.0
        var swing: float = sin(_walk_cycle) * 0.02       # 左右微摆 ±1.15°
        var bounce: float = abs(sin(_walk_cycle)) * 0.01  # 引擎上下弹跳
        if _sprite:
            _sprite.rotation = swing
        if _engine_glow:
            _engine_glow.position.y = 32.0 + bounce * 20.0
    else:
        reset_pose()
```

两个三角函数，一个控制旋转，一个控制位移，速度阈值 `> 10` 决定播放还是归零。没有关键帧，没有曲线编辑器，改手感直接调 `* 8.0`（频率）和 `* 0.02`（幅度）的系数。

## 引擎呼吸光效

独立于行走动画的另一个脉动循环：

```python
var _breath_timer: float = 0.0
func _update_engine_breathing(delta: float) -> void:
    _breath_timer += delta * 2.5
    var breath: float = 0.85 + sin(_breath_timer) * 0.15
    _engine_glow.modulate.a = breath    # 透明度 0.7~1.0
    _engine_glow.scale.x = breath       # 宽度同步呼吸
```

频率 2.5Hz，幅度 ±0.15，让引擎火焰看起来像在"呼吸"。和行走摆动用不同的计时器，频率不同步，视觉上更有机。

## 屏幕震动：衰减随机偏移

受伤或爆炸时触发屏幕震动：

```python
var _shake_strength: float = 0.0
var _shake_decay: float = 5.0

func apply_screen_shake(strength: float) -> void:
    _shake_strength = strength

func _update_screen_shake(delta: float) -> void:
    if _shake_strength > 0.01:
        var offset := Vector2(
            randf_range(-1, 1) * _shake_strength,
            randf_range(-1, 1) * _shake_strength
        )
        camera.offset = offset
        _shake_strength = lerpf(_shake_strength, 0.0, _shake_decay * delta)
    else:
        camera.offset = Vector2.ZERO
        _shake_strength = 0.0
```

核心原理：每帧给相机一个随机偏移，幅度按指数衰减。调大 `strength` 是重震（爆炸），调小是轻震（受击）。`decay = 5.0` 让震动约 0.3 秒消散。

## 受击闪光：Emission 材质切换

敌人受击时全身变白，不用帧动画，直接改材质参数：

```python
# hit_effect.gd
func flash_white(duration: float = 0.1) -> void:
    var mat := _sprite.material as ShaderMaterial
    if not mat:
        mat = ShaderMaterial.new()
        _sprite.material = mat
    mat.set_shader_parameter("emission_color", Color.WHITE)
    mat.set_shader_parameter("emission_strength", 2.0)
    await get_tree().create_timer(duration).timeout
    mat.set_shader_parameter("emission_strength", 0.0)
```

死亡时换成红色 emission + 缩小动画，同样用 `lerp` 在 `_process` 里驱动。

## 子弹拖尾

子弹的尾迹不用粒子系统，用一段渐隐的 Line2D：

```python
# bullet.gd
var _trail_points: PackedVector2Array = []
const MAX_TRAIL_POINTS := 8

func _process(delta: float) -> void:
    _trail_points.push_front(global_position)
    if _trail_points.size() > MAX_TRAIL_POINTS:
        _trail_points.resize(MAX_TRAIL_POINTS)
    _trail_line.points = _trail_points
```

每帧把当前位置插入数组头部，超过长度就裁剪。Line2D 自带宽度渐变，配合 0→1 的 alpha 衰减，效果和粒子拖尾几乎一样，但零粒子开销。

## 为什么不用 AnimationPlayer？

| | AnimationPlayer | 程序化 |
|--|--|--|
| 适用场景 | 复杂关键帧动画（角色骨骼、过场动画） | 简单循环效果（震动、脉动、摆动） |
| 迭代速度 | 打开编辑器调曲线 | 改一个数字 |
| 资源管理 | .tres 文件需要管理 | 纯代码，零外部依赖 |
| 运行时灵活性 | 播放/暂停/混合 | 可以和游戏状态实时联动 |
| 适用规模 | 大量动画需要复用 | 少量简单效果 |

这个项目的所有动画都是简单循环效果，没有骨骼、没有帧动画、没有过场。用 sin + lerp + randf 三个函数覆盖了全部需求。

> 程序化动画不是"穷人版动画系统"，是为特定问题量身定做的最简方案。
